# dx2-comfyui-s3-tools

ComfyUI custom node package for uploading generated videos to S3-compatible
storage (AWS S3, RunPod Network Storage, MinIO, etc.).

---

## Nodes

### DX2 Upload Video to S3 (`DX2UploadVideoToS3`)

Uploads a video file to an S3-compatible bucket after it has been generated by
your workflow.

#### Inputs

| Name | Type | Required | Description |
|---|---|---|---|
| `local_path` | STRING | тЬЧ | Explicit filesystem path to the video file. |
| `vhs_filenames` | VHS_FILENAMES | тЬЧ | Output of `VHS_VideoCombine` from ComfyUI-VideoHelperSuite. The last file in the list is used. |
| `job_id` | STRING | тЬЧ | Optional job/run ID inserted into the S3 key: `{prefix}/{job_id}/{filename}`. |
| `s3_key_prefix` | STRING | тЬЧ | Leading path for the S3 key (default: `videos`). |
| `enabled` | BOOLEAN | тЬЧ | Set to `false` to skip upload and return `"upload_skipped"` (default: `true`). |

At least one of `local_path` or `vhs_filenames` must be connected.
When both are provided, `local_path` takes priority.

#### Output

| Name | Type | Description |
|---|---|---|
| `upload_info` | STRING | S3 URI of the uploaded file, e.g. `s3://my-bucket/videos/job-1/output.mp4`. Returns `"upload_skipped"` when disabled. |

## Installation

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/mluciani/dx2-comfyui-s3-tools.git
pip install -r dx2-comfyui-s3-tools/requirements.txt
```

Restart ComfyUI after installation.

## Configuration

All credentials are read from **environment variables** тАФ nothing is hard-coded
in the workflow JSON.

| Variable | Required | Description |
|---|---|---|
| `S3_BUCKET` | тЬЕ | Destination bucket name. |
| `AWS_ACCESS_KEY_ID` | тЬЕ | Access key ID (or compatible credential). |
| `AWS_SECRET_ACCESS_KEY` | тЬЕ | Secret access key. |
| `S3_ENDPOINT_URL` | тЬЧ | Custom endpoint for S3-compatible stores, e.g. `https://s3.runpod.io`. Omit for standard AWS S3. |
| `S3_REGION` | тЬЧ | AWS region (default: `us-east-1`). |

### Example (RunPod)

```bash
export S3_BUCKET=my-runpod-bucket
export S3_ENDPOINT_URL=https://s3.runpod.io
export AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
export AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

## Wiring examples

### With VHS_VideoCombine

```
VHS_VideoCombine тЖТ vhs_filenames тЖТ DX2UploadVideoToS3
```

Connect the `VHS_FILENAMES` output of `VHS_VideoCombine` directly to the
`vhs_filenames` input. No intermediate selector node is required.

### With a plain file path

```
(any STRING node) тЖТ local_path тЖТ DX2UploadVideoToS3
```

## Error handling

| Situation | Behaviour |
|---|---|
| File not found | Raises `FileNotFoundError` with the resolved path. |
| Missing credentials | Raises `EnvironmentError` naming the missing variable. |
| Upload failure (AWS error / mid-transfer failure) | Raises `RuntimeError` with `s3://bucket/key` context. |
| `enabled` is `false` | Returns `"upload_skipped"` cleanly with no side effects. |
